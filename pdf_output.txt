Abstract 
The rapid growth of internet usage and evolving network protocols have necessitated
efficient and accurate methods for internet traffic classification. This project 
introduces an innovative approach, the Adaptive Hybrid Decision Tree Forest 
(AHDTF), designed to address challenges like high-dimensional data, imbalanced 
traffic patterns, and scalability. The AHDTF combines the strengths of decision tree 
classifiers with adaptive mechanisms such as feature weighting using mutual 
information, bootstrapping, and tree dropout to enhance robustness and 
generalization.
The proposed model emphasizes feature importance through a weighted combination
of mutual information and Gini impurity, ensuring that critical traffic attributes are 
prioritized during training. Additionally, the ensemble approach incorporates 
randomness in feature selection and bootstrapped datasets, effectively reducing 
overfitting while maintaining high classification accuracy. The inclusion of tree 
dropout introduces further diversity, preventing model reliance on a small subset of 
trees during predictions.
Experiments were conducted on a preprocessed internet traffic dataset (log2.csv), 
which contains features like port numbers, traffic volume, and protocol types, with 
the target column being classified into four distinct classes. The model achieved an 
impressive training accuracy of 99.79% and a testing accuracy of 99.78%, 
showcasing its generalization ability. The precision, recall, and F1-score metrics 
indicate outstanding performance across most classes, with the weighted average F1-
score reaching 1.00. However, performance on less-represented classes (e.g., Class 
3) highlighted the need for addressing class imbalance in future iterations.
The results underline the AHDTF's potential as a robust and efficient solution for 
real-world internet traffic classification. This work provides a foundation for future 
research in scalable traffic classification models, with potential applications in 
network security, resource allocation, and quality-of-service optimization.

2. Proposed Methodology
2.1 Overview of the AHDTF Model
The Adaptive Hybrid Decision Tree Forest (AHDTF) model is an ensemble machine
learning algorithm that combines the strengths of decision trees and the flexibility of 
adaptive learning mechanisms. The model aims to improve accuracy and robustness 
in classification tasks by integrating multiple decision trees with enhanced features 
like bootstrap sampling, feature selection, and a dynamic tree-dropping strategy. The
key idea behind AHDTF is to aggregate predictions from a diverse set of decision 
trees trained on random subsets of features and instances, enabling it to capture more
complex patterns in the data.
The AHDTF model consists of the following components:
1.Hybrid Decision Trees (HDTs): Each tree in the forest is an enhanced 
decision tree where the splitting criterion is based on a weighted combination 
of mutual information and Gini impurity. 
2.Feature Weighting: Each feature is assigned a weight based on its mutual 
information with the target variable. This weighting is used during training to 
prioritize more informative features. 
3.Tree Dropout: Not all trees are used during prediction. A percentage of trees 
are randomly dropped, ensuring that the ensemble does not overfit to the 
training data. 
4.Bootstrapping: Each tree is trained on a bootstrapped sample of the data, 
allowing the model to generalize better and reduce variance. 
This approach leads to a more adaptive and robust decision tree ensemble that can 
handle high-dimensional datasets effectively.
2.2 Key Components and Algorithm
The AHDTF model is built using two primary components: the Hybrid Decision 
Tree and the Adaptive Decision Tree Forest. Below is a breakdown of the key 
components and their associated algorithms.
Hybrid Decision Tree (HDT)
The Hybrid Decision Tree is a variant of the standard decision tree where the tree 
splitting criterion is customized. The algorithm follows these steps:
--- PAGE 6 ---
1.Feature Selection: At each node, the algorithm selects a feature based on the 
mutual information score with the target variable. This selection helps to 
prioritize features that are most informative for classification. 
2.Gini Impurity Adjustment: The traditional Gini impurity measure is adjusted
by incorporating the mutual information of the chosen feature, improving the 
splitting criterion to account for both the feature's informational value and the 
homogeneity of the resulting subsets. 
3.Tree Building: The decision tree is grown recursively, with each node being 
split based on the feature that maximizes the custom split score (a combination
of mutual information and Gini impurity). 
Adaptive Decision Tree Forest (AHDTF)
The Adaptive Decision Tree Forest is an ensemble of multiple Hybrid Decision 
Trees that collaborate to make predictions. The key steps in the algorithm are:
1.Feature Weighting: Before training, each feature’s importance is calculated 
based on mutual information with the target variable. The weights are 
normalized so that they sum to 1, making them suitable for probabilistic 
sampling. 
2.Bootstrap Sampling: Each tree in the ensemble is trained on a bootstrap 
sample (random sampling with replacement) of the training data. This ensures 
that each tree has a different subset of data, enhancing the diversity of the 
ensemble. 
3.Random Feature Selection: For each tree, a subset of features is randomly 
selected based on the calculated feature weights. The number of features to be 
selected is determined by the max_features parameter (e.g., square root of
the total features). 
4.Tree Training: Each tree is trained on its respective bootstrap sample with the
selected subset of features. 
5.Tree Dropout: During prediction, a fraction of trees are randomly dropped 
(controlled by the tree_dropout parameter) to prevent overfitting and 
encourage diversity in the ensemble's predictions. 
6.Prediction: The predictions of the active trees are aggregated. The final 
prediction is the class that is most frequently predicted across the selected 
trees. 
This methodology leverages both the advantages of decision trees (interpretability 
and simplicity) and the flexibility of ensemble methods (robustness and accuracy). 
The dynamic nature of the algorithm, including tree dropout and feature weighting,

high-dimensional datasets effectively.

3. Implementation
3.1 Dataset Description
The dataset used for this project contains 11 attributes and a target column called 
'Action', which is to be predicted. The attributes are as follows:
1.'Source Port': The source port of the connection. 
2.'Destination Port': The destination port of the connection. 
3.'NAT Source Port': The Network Address Translation (NA T) source port. 
4.'NAT Destination Port': The NAT destination port. 
5.'Bytes': The total number of bytes transmitted in the session. 
6.'Bytes Sent': The number of bytes sent from the source. 
7.'Bytes Received': The number of bytes received by the source. 
8.'Packets': The total number of packets exchanged in the session. 
9.'Elapsed Time (sec)': The duration of the session in seconds. 
10.'pkts_sent': The number of packets sent from the source. 
11.'pkts_received': The number of packets received by the source. 
The target variable 'Action' contains the following four classes:
1.allow: The action is allowed. 
2.deny: The action is denied. 
3.drop: The action is dropped. 
4.reset-both: The action is reset for both sides of the connection. 
These attributes and the target variable are used to build a machine learning model 
that classifies each session into one of these four classes. The dataset is divided into 
training and testing subsets, and a variety of machine learning algorithms are 
employed to analyze the relationship between the attributes and the target variable, 
ultimately predicting the 'Action' for new instances.

3.2 Model Configuration
The proposed model is an Adaptive Decision Tree Forest (AHDTF), which is an 
ensemble method leveraging multiple Hybrid Decision Trees. The configuration for
training the model is as follows:
Hybrid Decision Tree (HDT):
The Hybrid Decision Tree incorporates a custom splitting criterion that combines 
mutual information and Gini impurity:
1.Mutual Information: This measure quantifies the amount of information 
shared between each feature and the target variable. It is used to prioritize 
features that provide significant predictive power for the target class. 
2.Gini Impurity: This is a common metric for decision trees that measures the 
impurity or disorder of the target variable at each node. By using both mutual 
information and Gini impurity, the tree's splits are optimized for both 
informativeness and purity. 
The Hybrid Decision Tree is trained with these parameters:
• max_depth: Set to 10 to allow deeper trees that can capture complex patterns 
without overfitting. 
• min_samples_split: Set to 2, ensuring the tree can split until there are at least 
two samples at each node. 
Adaptive Decision Tree Forest (AHDTF):
The AHDTF model consists of an ensemble of multiple Hybrid Decision Trees, 
trained with the following configuration:
1.n_estimators: The forest contains 300 trees, providing stability and diversity 
in the ensemble's predictions. 
2.max_depth: Each tree has a maximum depth of 10, allowing the trees to 
model more intricate relationships. 
3.tree_dropout: A dropout rate of 20% is applied during prediction to prevent 
overfitting and improve generalization. 
4.max_features: The number of features considered for each tree split is 
determined by the square root of the total number of features ('sqrt'), a 
typical choice in ensemble learning. 
5.bootstrap: True enables bootstrapping, meaning each tree is trained on a 
random subset of the data with replacement, further improving diversity and 
reducing variance in the model.
