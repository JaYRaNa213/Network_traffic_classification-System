Network Telemetry Parameters

The prediction engine requires 11 specific network telemetry parameters to calculate the Mutual Information feature weights and execute deterministic tree routing. Below is a detailed technical explanation of the input vector required for classification.

Source / Destination Ports

The original logical network ports utilized by the host to transmit (Source) and receive (Destination) the traffic flow.

NAT Source / Destination Ports

Network Address Translation (NAT) ports. These are the mapped ports translated by the firewall or router, critical for detecting masking anomalies.

Bytes (Total, Sent, Received)

The absolute payload volume of the network session. Analyzing the ratio of bytes sent vs. bytes received helps identify data exfiltration attempts.

Packets (Total, Sent, Received)

The discrete number of data fragments routed through the network. Packet counts relative to byte sizes reveal the protocol nature (e.g., streaming vs polling).

Elapsed Time

The total duration of the network connection in seconds. Sustained long-duration connections are a primary vector evaluated by the Adaptive Forest.

Comprehensive Model Architecture

The core of this platform is powered by a custom-built Adaptive Decision Tree Forest. Unlike traditional ensemble algorithms (e.g., standard Random Forests) which treat all features equally, our architecture dynamically evaluates network traffic using Mutual Information scoring prior to any node splits.

The model is trained on 11 critical network telemetry features (Source Port, Dest Port, NAT properties, Bytes/Packets Sent and Received, and Elapsed Time). By analyzing these feature distributions, it deterministically classifies the network action into four primary categories: Allow, Drop, Deny, and Reset-both.

Feature Weighting Pipeline Dominant packets don't mask subtle zero-day anomalies because all 11 features are weighted intelligently using Mutual Information metrics before tree aggregation.
Deterministic Tree Consensus The ensemble operates via active tree drop-out and bootstrap aggregation, yielding highly deterministic confidence scoring rather than pseudo-random probabilities.
High-Throughput Inference The prediction engine is optimized for sub-100ms classification latency without sacrificing the depth or complexity of the 300+ estimators in the forest.

About The Project

Trained on internet firewall traffic data (log2.csv), this model classifies network flows into ALLOW, DENY, DROP, or RESET-BOTH categories. Engineered to combat the limitations of standard ensemble learning methods, it aggregates predictions via active tree drop-out logic to ensure deterministic confidence scoring and resilience against data poisoning.
